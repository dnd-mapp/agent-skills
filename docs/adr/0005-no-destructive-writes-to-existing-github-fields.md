# No destructive writes to existing GitHub Project or org fields in update mode

When `configure-issue-tracker` runs in update mode and the user changes options on a field that already exists in a GitHub Project or at the org level (e.g., adding a new Issue Type to an existing single-select field), the skill updates `docs/agents/` config locally to reflect the new intended state but does not touch GitHub. Instead, it produces explicit guidance describing exactly what the user needs to change manually in the GitHub UI or via `gh` CLI and how. The alternative — deleting and recreating the field with the new options — was rejected because it destroys all existing field values on Project items, which is a destructive action against shared team data. Silently diverging config from GitHub without surfacing the gap was also rejected because agents would operate on stale config.
