# Local Markdown files as the initial issue storage implementation

The `configure-issue-tracker` skill starts with local Markdown files at `docs/planning/issues/` as its first storage backend. This is a phased approach, not a rejection of hosted trackers. Local files were chosen as the foundation because they require no API credentials, keep issues versionable and co-located with the code, and let the skill's interview and config structure be validated before adding remote complexity. The trade-off is that the local backend lacks hosted-tracker features (notifications, assignment workflows, cross-repo visibility) and is not suitable as a team's primary tracker in multi-developer contexts. **GitHub Issues and GitHub Issues + Projects are now supported as Tracker Backends** (see ADRs 0005 and 0006). Jira support is planned for a future iteration.
