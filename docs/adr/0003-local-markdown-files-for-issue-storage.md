# Local Markdown files as the initial issue storage implementation

The `configure-issue-tracker` skill starts with local Markdown files at `docs/planning/issues/` as its first storage backend. Hosted trackers (GitHub Issues, Linear, Jira) are planned for a future iteration — this is a phased approach, not a rejection of hosted trackers. Local files were chosen as the foundation because they require no API credentials, keep issues versionable and co-located with the code, and let the skill's interview and config structure be validated before adding remote complexity. The trade-off is that the local backend lacks hosted-tracker features (notifications, assignment workflows, cross-repo visibility) and is not suitable as a team's primary tracker in multi-developer contexts.
